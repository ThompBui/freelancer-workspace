# Internationalization (i18n)

## UI languages

| Code | Language | Priority |
|------|----------|----------|
| `en` | English | Default (global) |
| `zh-CN` | 简体中文 | High |
| `ja` | 日本語 | High |
| `es` | Español | High |
| `pt` | Português | High |
| `ko` | 한국어 | Bonus |
| `vi` | Tiếng Việt | Bonus |

## Files

- `src/i18n/en.js` — source of truth (English)
- `src/i18n/*.js` — other locales (same key tree)
- `src/i18n/index.js` — merge + `t()` helper (missing keys fall back to English)
- `src/i18n/localeMeta.js` — language picker labels

## Demo / dynamic content

Project names, client names, messages in mock data remain **Vietnamese** by design. Status values in data stay Vietnamese keys (`Đang làm`, `Chậm tiến độ`, …); the UI maps them through `demoStatus.*` keys in `src/utils/demoStatusLabels.js` so badges and filters follow the selected language.

## Adding a string

1. Add key to `en.js`
2. Mirror in `vi.js`, `zh-CN.js`, `ja.js`, `es.js`, `pt.js`, `ko.js` (or rely on English fallback until translated)
3. Use `t('section.key')` in components
